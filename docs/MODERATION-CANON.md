# MODERATION-CANON.md — ねがお審査基準書 v1.0（確定）

> 毎日の審査（approve/reject）の判断基準。目的は「速く・一貫して・安全に」。
> 将来のAI事前フィルタの教師定義、moderation系skillの参照元を兼ねる。
> 改訂は中西の判断のみ。グレー事例は§6のログに貯め、月次で本書に還元する。
> v1.0確定事項: D1=穏やかまで通す / D2=病気・怪我系は厳格に除外 / D3=メモリアル要素は除外 /
> D4=配達せず記録 / D5=迷ったらreject

---

## 0. 大原則

1. **受信者の一通を守る。** 受け取る側にとって、きょうのねこだよりは一日一枚しかない。
   その一枚が不快・不安・期待外れであるコストは、一枚の除外のコストよりはるかに大きい。
2. **迷ったら除外（reject）。** 除外しても投稿者のアルバムには残り、
   投稿者への個別通知・理由説明は行わない（規約A-2）。除外は静かで、コストが低い。
3. **審査は品評ではない。** 写真の上手下手を裁かない。基準は
   「知らない誰かの今夜の一通として差し支えないか」だけ。生活感・多少のブレは味。
4. **基準は人間が定義し、AIが近似する。** 将来のAI事前フィルタは本書を
   三値（approve / reject / 要人間）で近似する。機械が判定しやすいように
   基準を狭めることはしない。

## 1. 即approve（考えなくてよいもの）

- 寝ている・目を閉じて休んでいる猫が主役に写っている
- 起きているが穏やかに休んでいる（香箱・まどろみ・薄目・丸くなっている）
- 多少のブレ・暗さ・構図の乱れ（猫と分かる範囲）
- 生活感のある背景（床・洗濯物・散らかった部屋）
- 複数の猫
- 人の手・膝・足など、個人を特定できない写り込み
- ケージ・キャリー内で穏やかに休んでいる（通院・防災は暮らしの事実）
- 布団・毛布から一部だけ見えている（耳・しっぽ等。猫と認識できることが下限）

## 2. 即reject（考えなくてよいもの）

- 猫が写っていない／猫がほぼ判別できない（極端な暗さ・ブレ）
- 明らかに活動中（立って歩く・遊んでいる・食べている・完全に覚醒したカメラ目線）
- 猫以外の動物が主役（犬・うさぎ等。ねてるねこである）
- イラスト・ぬいぐるみ・フィギュア・画面の再撮影・印刷物の撮影
- AI生成の疑いが強いもの
- 人の顔が識別できる写り込み（規約A-2(2)）
- 個人情報の写り込み（宛名・書類・画面上のメールや氏名・車のナンバー等）
- 性的・暴力的・グロテスクな要素を含むもの
- 商用・宣伝的な要素（商品パッケージの意図的配置、QRコード、文字入れ加工）
- **病気・怪我・衰弱が見て取れるもの**（外傷・点滴・術後の痛々しさ・極端な痩せ）
  — 受信者に持ち帰れない不安を渡さない。投稿者の日常の否定ではなく、配達しないだけ
- **メモリアル的な要素**（花・ろうそく・額縁・遺影の雰囲気）
  — 死の想起の遮断はプロダクト全体の禁忌と同根。ただし単なる花瓶の写り込みまで疑わない

## 3. グレーゾーン判定表

| # | ケース | 判定 | 根拠 |
|---|---|---|---|
| G1 | エリザベスカラー・術後服だが、痛々しさがなく穏やかに休んでいる | approve可・迷えばreject | 避妊去勢後は日常の範囲。「痛々しさ」が主印象なら§2へ |
| G2 | 子猫すぎて衰弱と区別がつかない | 迷えばreject | 元気な子猫の寝顔はapprove |
| G3 | 屋外・外猫・地域猫らしき猫 | approve | 「うちのこ」限定を審査で強制しない（判定不能）。劣悪・危険な環境が主印象ならreject |
| G4 | 狭すぎる・汚れが目立つケージ、拘束的な状況 | reject＋記録 | 虐待とは断定しないが配達もしない。§5の記録対象 |
| G5 | 覚醒と休息の中間で判定しがたい | reject | 大原則2。グレーログに1行残す |

## 4. 判断フロー（日課用・上から順）

```
猫が写っている? ──no→ reject
猫が主役?（§2の即reject要素なし） ──no→ reject
穏やかに休んでいる? ──no→ reject
不安・死の想起を渡さない? ──no→ reject（G4は記録）
→ approve
```

- 1枚あたり目安3秒。10秒迷ったらグレーログ（§6）に書いてreject
- 審査時刻: 毎日19:00〜19:30【運用で確定。**時刻の正は本節**——ROADMAP等の記載は参照】。20:00便の締切は19:55

## 4.5 配達プールの考え方

- 配達プールはストック制。毎日リセットされず、過去にapproveされた写真は在庫として残る。
- 候補は3層Tierで扱う。Tier1は今日の投稿、Tier2は過去在庫、Tier3はadmin stock。
- 同じTier内では、`delivery_count` が少ない写真を優先し、その上で受け取るユーザーごとにシャッフルする。
- 同一写真は同一人に二度届かない。再配達防止は審査判断ではなく配達ロジック側の責務。
- **並び順・重複防止の実装詳細の正は配達spec（`docs/specs/spec-v1.3.md` §3）。本節は審査者向けの要約**（配達ロジック変更時はspec側を改訂し、本節は要約として追随させる）。
- approveは「今夜だけ配ってよい」という判定ではなく、「配達在庫に永住させてよい」という判定である。

## 5. エスカレーション（rejectで終わらせないもの）

- **虐待の疑い**（複数回のG4、外傷の反復、状況の悪化）:
  reject＋当該anonymous_idの投稿履歴を確認し、日付・内容を `docs/moderation-log/` に記録。
  通報の判断（動物愛護管理法・自治体窓口）は中西が行う。単発では動かない
- **同一ユーザーの反復違反**（§2該当を3回以上）: 記録の上、対応（案内DM・制限）を個別判断
- **photo_reports（ユーザー通報）が付いた配達済み写真**: 24時間以内に目視し、
  妥当なら遡ってreject（以後の配達停止）。通報者への個別返信はしない

## 6. グレーログ運用

- 迷った1枚ごとに1行: 日付 / 内容の一言 / 判定 / 迷った理由
- 月次で見返し、繰り返し出る型を§1〜3に昇格させて本書を改訂
- このログがそのまま、将来のAI事前フィルタ（猫検出＋NSFW＋本書ルール、三値出力）の
  教師データ仕様になる

## 7. 本書が裁かないもの

- 写真の巧拙・美しさ（品評はしない）
- 猫種・毛色・年齢による選別（すべての猫は等しく配達される）
- 投稿者のアルバムの中身（審査対象は配達プール入りだけ。本人の記録には一切触れない）
