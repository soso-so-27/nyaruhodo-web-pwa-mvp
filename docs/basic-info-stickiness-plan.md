# うちのこ > 基本 情報設計メモ

## 目的

`基本` は、単なるプロフィール入力欄ではなく、`この子がどんな子か` を少しずつ知っていく場所にする。

`記録` は、日々の出来事や写真がたまる場所として維持する。

- `基本`: この子の輪郭
- `記録`: この子との時間

この2つを混ぜない。

## 基本タブ

目的: `この子がどんな子か` を残す場所。

性質:

- 比較的変わりにくい
- この子を見るときの前提になる
- 写真や記録を見返すときの文脈になる
- 入力して終わりではなく、将来的にホーム、記念、年まとめ、物販導線に返せる

基本タブ内の将来構成:

1. 代表写真
2. 基本情報
3. 呼び名・この子らしさ
4. ケア情報

## 記録タブ

目的: `この子と何があったか` を残す場所。

性質:

- 日々増える
- 時系列でたまる
- 写真、足あと、記念、年まとめに使う
- 変化や積み上がりを見る

記録タブに属するもの:

- `ねがお`
- `この子の写真`
- `とどいたねこだより`
- 足あと
- 記念
- 年まとめ
- その日に起きた出来事

## A. 代表写真

役割: この子を見分けるための写真。

方針:

- `この子の写真` と `ねがお` の両方から選べる
- 端末から直接選べる
- 自動表示に戻せる
- 代表写真は `catGalleryPhotos` そのものではない
- `ねがお` を選んでも `catGalleryPhotos` にコピーしない
- `この子の写真` 100枚枠を消費しない

将来の使い道:

- うちのこページ上部
- 猫切替
- 年まとめ
- カード、カレンダー、小さな本などの物販導線

## B. 基本情報

現在あるプロフィール情報:

- 名前
- 家族になった日
- 誕生日
- 性別
- 猫種・タイプ
- 毛色

目的: この子を見るときの基本的な文脈を作る。

保存先:

- `catProfiles[].basicInfo`
- remote syncでは `cats.basic_info` JSON に含める
- DBカラムは追加しない

将来の返し方:

- `今日は家族になって3年と278日`
- `誕生日まであと9日`
- `今年も誕生日を迎えました`
- `この毛色の子らしい写真が増えてきました`

注意:

- 医療、診断、品種判定のようには扱わない
- 猫種や毛色は、正確な分類よりも飼い主が思っている呼び方を優先する

## C. 呼び名・この子らしさ

粘着性を高める本命ブロック。P1として最小項目を実装済み。

候補:

- よく呼ぶ名前
- 呼ぶと来る
- 耳だけ動く
- しっぽだけ動く
- 反応しないけど聞いている気がする
- 好きな場所
- 好きな寝場所
- 好きな遊び
- なでられると好きなところ
- 苦手なこと

目的:

- 日々の出来事ではなく、基本タブ内の `この子らしさ` として扱う
- 飼い主が「この子はこういう子」と思っていることを残す
- アプリが後からこの子らしい文脈として返せる材料にする

保存先:

- `basicInfo.personality.callName`
- `basicInfo.personality.favoritePlace`
- `basicInfo.personality.favoritePlay`
- `basicInfo.personality.favoriteTouch`
- `basicInfo.personality.dislikes`

将来の返し方:

- `むぎって呼ぶと、耳だけ動く日があります`
- `ソファが好きな場所でしたね`
- `なでられると好きなところ、また同じですね`
- `今日は好きな遊びの写真も残してみますか`

UIでは研究名や専門用語を出しすぎず、猫あるあるとして自然に見せる。

## D. ケア情報

新しい `ケア` タブは作らず、基本タブ内の将来候補として扱う。

目的: 健康管理アプリにするためではなく、飼い主がこの子を見返すための最低限のケアメモ。

保存先:

- `basicInfo.care.weightKg`
- `basicInfo.care.weightMeasuredDate`
- `basicInfo.care.vetClinic`
- `basicInfo.care.careNote`

見せ方:

- `健康管理` ではなく `ケア情報` または `ケアのメモ`
- アプリが医学的な判断をしない
- 獣医師に相談する前提の控えにする

P1として実装済み:

- 体重
- 最後に測った日
- かかりつけ病院
- 気をつけること

P2候補:

- ワクチン・予防メモ
- 避妊・去勢
- マイクロチップ
- 保険メモ

体重の扱い:

- 基本タブでは `最新の体重` と `最後に測った日` だけを持つ
- 体重履歴やグラフは公開前には実装しない
- 将来的に記録タブへ `体重を測った` 履歴として出せる
- アプリが太りすぎ、やせすぎを判定しない

ワクチン・予防メモの扱い:

- ワクチンに特化しない
- ケア情報の一部として扱う
- 接種要否をアプリが判断しない
- 次回時期を自動判定しない
- 獣医師に言われた予定日やメモを任意で残すだけ
- 証明書写真アップロードは公開前には実装しない

## 猫の生態・研究知見から見た設計理由

- AAFP/ISFMの猫の環境ニーズガイドラインでは、猫の快適さは環境、健康、行動と結びついており、安心できる場所や予測しやすい環境が重要とされている。`好きな場所` や `好きな寝場所` は、この子らしさとして残す価値がある。  
  参照: [AAFP/ISFM Feline Environmental Needs Guidelines](https://catvets.com/resource/aafp-isfm-environmental-needs-guidelines/)
- 猫が自分の名前を他の語から区別できることを示す研究がある。`よく呼ぶ名前` や `呼んだときの反応` は、飼い主との関係性として記録価値がある。  
  参照: [Domestic cats discriminate their names from other words](https://www.nature.com/articles/s41598-019-40616-4)
- WSAVAの栄養評価資料では、体重やボディコンディションなどを継続的に見る考え方が示されている。ねてるねこでは診断ではなく、変化に気づくための控えとして扱う。  
  参照: [WSAVA Global Nutrition Guidelines](https://wsava.org/global-guidelines/global-nutrition-guidelines/)
- WSAVAのワクチンガイドラインは地域や状況で判断が変わることを前提にしている。アプリが接種要否を判断せず、獣医師から聞いた予定やメモを控えるだけにする。  
  参照: [WSAVA Vaccination Guidelines](https://wsava.org/global-guidelines/vaccination-guidelines/)

## 守ること

- 既存の `記録` タブの役割を変えない
- 新しい `ケア` タブは作らない
- `ねがお` は `ownSleepingPhotos`
- `この子の写真` は `catGalleryPhotos`
- `代表写真` は別概念
- 代表写真は `この子の写真` と `ねがお` の両方から選べる
- `ねがお` を代表写真にしても `catGalleryPhotos` にコピーしない
- `この子の写真` 100枚枠を消費しない
- `catGalleryPhotos` はねこだより候補に混ざらない
- `slot_slug="__cat_gallery"` は通常collectionに混ざらない
- 既存の同期、保存、削除処理を壊さない
