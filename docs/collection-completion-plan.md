# Collection Completion Plan

## Goal

`/collection` は、猫の写真をただ保存する場所ではなく、「次に見つけたい姿があり、撮ると棚が埋まり、あとで見返したくなる場所」にする。

## Current Completion Line

### 1. あつめる

- 目的: 今日撮るものを迷わず決める。
- MVP状態: `今日の見つけたい姿` と `次に残したい姿` を表示。
- 次の仕上げ:
  - 今日カードを写真追加の主導線として維持する。
  - 候補は多すぎると散るため、最大4件のままにする。
  - 追加後は「見つけた」フィードバックを出す。

### 2. アルバム

- 目的: ためた姿を棚として見返す。
- MVP状態: カテゴリ別グリッド、写真あり/なし、今日バッジ。
- 次の仕上げ:
  - 空カードは背景を主張させず、名前とシルエットを見やすくする。
  - 写真ありカードはラベルと枚数だけにする。
  - カテゴリ名は増やしすぎない。

### 3. シェア

- 目的: 将来のアプリ内共有フィードの入口を作る。
- MVP状態: 自分の一枚 + 次に残したい候補。
- 次の仕上げ:
  - 共有済みのように見せない。
  - 候補カードは「撮るための入口」として扱う。
  - 将来は `self_photo` / `shared_photo` / `suggested_candidate` を同じカードモデルで差し替える。
