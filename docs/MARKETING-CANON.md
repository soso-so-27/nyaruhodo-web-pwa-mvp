## 0. 一文の憲法

**ねてるねこは、静かなアプリである。宣伝もまた、静かでなければならない。**

発信そのものがプロダクトの一部である。煽る告知は、アプリを一度も開かせる前に世界観を裏切る。

---

## 1. トーン原則（静けさの五箇条）

1. **煽らない** — 緊急性・限定性・カウントダウンで行動を迫らない。
   「急がずに作っています」が公式の速度感。
2. **数字を誇示しない** — フォロワー数・DL数・MAU・ランキングを対外的に語らない。
   数字は内部レポート専用。過去のバイラル実績（再生数）にも言及しない。
3. **broadcastしない** — 「拡散希望」「シェアお願いします」を書かない。
   届く人に届けばよい、という姿勢を崩さない。
4. **約束しない** — リリース日・機能追加を事前確約しない。
   予告は「できあがったら、お知らせします」の形式まで。
5. **打ち明ける** — 宣言・発表ではなく、打ち明け・報告の文体。
   主語は「私（中西）と、あめとむぎ」。企業の声を作らない。

> 語彙ロックの正は `AGENTS.md`（2026-07-07一本化）。本表は対外発信への適用表であり、
> 差分が生じたらAGENTS.mdが勝つ。

| 使う | 使わない |
|---|---|
| ねがお | 寝顔写真・スリーピングショット |
| ねがおを とる | 撮影する・アップする・投稿する（アプリ内動作の説明時） |
| ねこだより | 通知・配信・フィード |
| ひらく | 開封する・チェックする |
| とどく | 配信される・プッシュされる |

- 「毎日」より「一日一枚」。頻度ではなく上限として語る。
- **2匹の呼称**: あめ（めー）・むぎ（むー）。IGでの通称はフォロワーの慣習に合わせ、
  中西の普段の呼び方を優先する（本CANONでは固定しない）。

## 3. 禁止表現（機械検査対象）

以下を含む文面は、理由を問わず差し戻す。

- **「一生」およびねこの死を想起させる語**（一生・最期・見送る・虹の橋 等）
  ※プロダクト内と同一の禁止。追悼文脈は本CANONの対象外（発信では扱わない）
- **煽り語**: 今だけ・お早めに・残りわずか・見逃すな・必見・バズ
- **誇示語**: ◯◯人突破・◯◯万再生・ランキング・話題の・大人気
- **依頼語**: 拡散希望・シェアお願いします・フォローお願いします
- **ゲーミフィケーション語**: 連続記録・ストリーク・ポイント・ランク
- **絵文字**: 原則使用しない（IGキャプション含む）
- **報酬型リファラルを想起させる表現**: 紹介で◯◯もらえる 等（機能として存在しないし、作らない）

## 4. IG運用規範（@ame.to.mugi）

### 4.1 アカウントの性格
- **「雨と麦」— あめとむぎ、2匹のアカウントのまま運用する。アプリ宣伝アカウントに転換しない。**
- フォロワーが待っているのは2匹。アプリは「2匹の寝顔から生まれたもの」として従属的に登場する。
- **アカウント名がアプリの起源の物語そのもの**（雨と麦の寝顔を撮りためて生まれた）。
  この接続は明示的に語ってよい。
- 投稿写真の主役は2匹どちらでも、2匹一緒でもよい。登場比率の管理はしない
  （アプリ側の複数猫セーフ設計と同じ思想: 「うちのこ」は全員が主役）。

### 4.2 構成比と頻度
- **2匹の日常（ねがお中心）7 : build in public 3**
- 頻度: **週2〜3投稿**。毎日投稿しない（アプリが一日一通であることと矛盾するため）。
- ストーリーズ: 開発の断片・途中経過を流す場。揮発（24h）を活かし、ラフでよい。
- フィード: 定着・宣言の場。

### 4.2.1 被写体の幅
- 穏やかな休息、寝る前後、まどろみ、起きているが静かに過ごしている様子は、日常7割の中に含めてよい。
- 完全覚醒のカメラ目線、遊び動画、動きの強い場面は量産しない。出す場合は build in public と合わせて全体の3割枠に収める。
- ねてるねこへの接続は「かわいい瞬間の量」ではなく、「一緒に暮らしている時間の見方」に置く。

### 4.3 リール
- **リールは使ってよい。** 本アカウントの実績あるリーチ手段であり、
  過去のバイラル（日常の情景）は煽らずに届いた前例である。
- 条件: **日常の延長のみ**。ねがお・2匹の関係・静かな情景。
- 禁止: トレンド音源への便乗、量産（週1本まで）、テロップ煽り
  （「最後まで見て」等）、再生数狙いの企画もの。
- 音: 無音・環境音・静かな音源を基本とする。

### 4.4 build in public の書き方
- 進捗数字・スクショの羅列ではなく、**「なぜ作っているか」の断片**を書く。
- 技術詳細より、2匹との生活の中でどう考えたか。
- 失敗・迷いも書いてよい（打ち明けの文体と整合する）。

### 4.5 やらないこと
- ストーリーズ連投による露出稼ぎ
- トレンド便乗リール・リール量産
- 他アカウントへの営業的コメント回り
- フォロワー数・再生数の推移への言及

## 6. KPIの語法（内部レポート用）

- 個人紹介の指標は **k（口コミ係数）ではなく継続率**。kは乗数にすぎず量を作れない、が確定認識。
- 量の指標は外部流入（将来: 保護猫団体welcome経由）。ただし団体向け機能は未実装のため、団体施策は本CANON v0.2 の対象外。実装後に §7 として追記する。
- IG実測値と見立ては `docs/weekly/2026-07-baseline.md` へ分離（2026-07-07）。
  数値の更新はweekly側でのみ行い、本CANONは語法だけを持つ。
  レポートは見立てとの差分で語り、絶対数の大小で一喜一憂しない。

## 8. 自動化の境界（Codex skillへの委任範囲）

### 委任する（定型）
1. 投稿ドラフト生成（写真＋メモ → キャプション案2種＋チェックリスト結果）
2. 禁止表現・語彙ロックの機械検査
3. 週次マーケレポート（数字入力 → 定型フォーマット出力）
4. コンテンツ在庫の棚卸（撮影済ねがお ↔ 投稿済の対応表）

### 委任しない（判断）
- 投稿する/しないの決定、投稿タイミング
- キャプションの最終文言（**中西が最終判断。Codexの出力は常に「案」**）
- 団体・個人との対話、DM返信
- 本CANON自体の変更

### skill運用規則
- すべてのマーケskillは本CANONを読み込んでから実行する。
- 出力が §3 禁止表現に抵触した場合、skillは出力を破棄し違反箇所を報告する。
- CANON改訂は中西の指示によってのみ行い、改訂履歴を本文書末尾に残す。

## 9. 付録: 参照アカウントの使い方

参照アカウントは鏡として使う。数字のベンチマークにしない。フォロワー数、再生数、投稿頻度、成長率を目標値に置かず、「どの距離感・言葉・余白がねてるねこに近いか」を見る。
