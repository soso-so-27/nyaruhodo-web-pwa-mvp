# Rich Motion Direction

8時に届く封筒と、ねこだよりページの写真展開をリッチ化するための方向性メモ。

## 目的

このアプリでモーションを入れる目的は、単なる装飾ではなく、次の3つに絞る。

- 8時に届いたものを開く瞬間を、毎日少し楽しみにできる体験にする
- ねこだよりに写真がたまる感覚を、文字ではなく直感で伝える
- 写真を押す、月を変える、届いた/おくったを切り替える時に、画面のつながりを壊さない

## ツール方針

### 採用軸

主軸は Motion for React にする。

理由:

- 既に `motion` が導入済み
- ホームの封筒開封で使っている
- React state、写真枚数、月、保存状態と直接つなげられる
- iPhone実機で段階的に軽量化しやすい
- `prefers-reduced-motion` に対応しやすい

Figma Motion は、実装本体ではなく、動きの検証・タイミング調整・デザイナー共有に使う。

Rive / Lottie は、将来的に固定演出パーツとして使う余地を残す。ただし、現時点では次の理由で主軸にしない。

- ユーザー写真、写真枚数、月切り替えなどの動的状態はReact側の方が扱いやすい
- 素材品質とアニメーション設計が弱いと、どのツールでも失敗する
- Riveは本格導入するなら高品質レイヤー素材と明確なstate machine設計が先に必要

## 1. 8時に届く封筒

### 現状

- `HomeEnvelopeMotionArt.tsx` で Motion for React を使用している
- 12枚のPNGレイヤーを時間差で動かしている
- 開封時間は `HOME_ENVELOPE_OPEN_MS = 2200`
- 既に `prefers-reduced-motion` 時の短縮分岐がある

### 問題

- 動きの気持ちよさより、レイヤーを順に表示している印象が強い
- 写真が出る必然性、蝋が割れる納得感、光の報酬感がまだ弱い
- 画像素材の品質に演出全体が引っ張られる
- 開いた後の写真表示とのつながりが薄い

### 目指す体験

「封筒をタップしたら、少し重みが返ってきて、蝋が割れ、紙が開き、光と一緒に写真が出てくる」。

派手な紙吹雪ではなく、紙・蝋・光・写真の質感でドーパミンを出す。

### 推奨タイムライン

全体は 2600ms 目安。現状の 2200ms は少し短いので、気持ちよさ優先で伸ばす。

| 時間 | 内容 | 意図 |
| --- | --- | --- |
| 0-120ms | タップで封筒が少し沈む | 押した感 |
| 120-360ms | 蝋が小さく弾む、影が濃くなる | 開く前のため |
| 360-720ms | 蝋が左右に割れる、細かい欠片が落ちる | 開封の納得感 |
| 620-1280ms | フラップがゆっくり起きる | 主演出 |
| 920-1500ms | 内側の光が漏れる | 報酬感 |
| 1200-2100ms | 写真カードが奥から出る | 届いたものの発見 |
| 1900-2600ms | 写真が定着し、影と光が落ち着く | 余韻 |

### レイヤー要件

封筒素材は次のレイヤーを個別に持つ。

- 背景影
- 封筒本体
- 内側パネル
- フロントポケット
- 上フラップ
- 蝋 intact
- 蝋 left
- 蝋 right
- 蝋 crumbs
- 内側 glow
- 写真カード placeholder
- 紙粉 motes

今のレイヤー構造は方向性として正しい。次に改善するべきは、レイヤー数ではなく素材の質とタイムライン。

### 実装方針

- `HomeEnvelopeMotionArt` は残す
- `HOME_ENVELOPE_OPEN_MS` を 2600ms 前後に再調整する
- `transition.times` を演出単位で分ける
- 写真カードは最初からDOMに存在させ、開封後に実写真へつなげる
- 低スペック/iPhoneでは blur と filter を抑える

### 成功基準

- タップした瞬間に押せた感じがある
- 開封が一瞬で終わらず、何が起きたか見える
- 蝋が割れたことが理解できる
- 写真が封筒から出たと感じる
- 開封後の写真表示に視線が自然に移る
- iPhoneでカクつかない

## 2. ねこだよりページの写真展開

### 現状

- `CollectionPage.tsx` の `MainichiMonthBoard` が月ごとの写真ボードを描画している
- `MAINICHI_PASTE_MOTION_CSS` で貼られる動きがある
- 写真配置は `getMainichiBoardPhotoLayout`
- 枚数が多い時は `getMainichiBoardVisiblePhotos` で表示制御している
- 写真タップ後はフルスクリーンビューアへ進む

### 問題

- CSS keyframes中心なので、月切り替え・写真追加・フルスクリーン遷移とのつながりが弱い
- 写真が「たまっている」より「ランダムに置かれている」に見えるリスクがある
- 枚数が多い時に、なぜ省略されるかが直感的に伝わりにくい
- 届いた/おくった切り替え時の体験が、感情よりUI操作に見える

### 目指す体験

「その月に集まった写真が、机の上に自然に広がる」。

カレンダーではなく、アルバムでもなく、届いた写真の束を開いて眺める感覚。

### 枚数別の見せ方

#### 0枚

- 写真枠を増やさない
- 空の台紙を強く見せない
- ほんの小さな封筒/紙片だけ置く
- 行動導線は控えめにホームへ戻す

目的: 入れられなかった日を責めない。

#### 1-3枚

- ほぼ等身大で、写真を主役にする
- 動きは少なく、1枚ずつ置かれる
- 余白を怖がらない

目的: 少ない枚数でも寂しく見せない。

#### 4-12枚

- 写真が中央から軽く広がる
- 主役写真を1-2枚作る
- それ以外は奥行きとして支える

目的: たまってきた感覚を出す。

#### 13-32枚

- すべてを同じ強さで見せない
- 主役3-5枚 + 背面の束で密度を出す
- 省略ボタンではなく「束の奥にもある」表現にする

目的: 多いほど嬉しいが、読みにくくしない。

#### 33-62枚

- 写真を全部同時に主役表示しない
- その月の代表写真を前面、残りは厚みとして表現
- 開くと日別/束ビューに進む

目的: 上限まで増えても破綻しない。

## 3. おくった / とどいた の切り替え

### 現状維持したいこと

- この2面は必要
- カレンダー化は避ける
- 文字説明に寄せすぎない

### 改善方針

タブを「ボタン感の強い切り替え」ではなく、同じ台紙の表裏を返す感覚に寄せる。

- おくった: 自分が残した写真
- とどいた: 8時に受け取った写真

切り替え時は、写真が全部消えて再表示されるのではなく、同じ月の束が裏返る/差し替わるようにする。

Motion for Reactでやるなら:

- `AnimatePresence` で出入り
- active side変更時に写真を少し後ろへ引く
- 180-260ms 遅れて新しい面の写真が前に出る
- 月とタブは動かしすぎない

## 4. 写真タップからフルスクリーン

### 目指す体験

写真を押したら、その写真がそのまま大きくなる。

いきなり別画面に飛ぶのではなく、触った写真とビューアが同じものだと分かることが大事。

### 実装候補

- 第1候補: Motion for React `layoutId`
- 第2候補: 現在の `MainichiMorphSource` を強化
- 第3候補: View Transitions API

まずは既存の `MainichiMorphSource` を活かしつつ、Motion導入で段階的に置き換える。

## 5. 実装順序

### Phase 1: 仕様固定

- このドキュメントをもとに、封筒とねこだよりの動きを2本に分ける
- それぞれの成功基準をスクショ/動画で確認する

### Phase 2: ねこだよりをMotion化

- `MainichiBoardPhotoCard` を `motion.button` に置き換える
- `MAINICHI_PASTE_MOTION_CSS` の責務を減らす
- 写真枚数別の `initial` / `animate` / `transition` を作る
- side切り替えを `AnimatePresence` でつなげる
- 写真タップ時の拡大導線を強化する

### Phase 3: 封筒開封の再設計

- タイムラインを 2600ms 前後にする
- 蝋/フラップ/写真カード/光の順序を再調整
- 開封後の写真への接続を強くする
- iPhone実機でカクつきを確認する

### Phase 4: Figma Motion検証

- Figma Motionで封筒開封のタイミングだけ再現する
- Dev ModeのMotion codeを参考に、React側の数値を調整する
- 完成品の持ち込みではなく、動きの採寸に使う

## 6. 検証

必ず見るもの:

- iPhone Safari
- ホーム画面追加後のPWA
- 朝/昼/夕方/夜背景
- 写真 0, 1, 3, 6, 12, 20, 40, 62枚
- おくった/とどいた切り替え
- 月切り替え
- 写真タップからフルスクリーン
- `prefers-reduced-motion`

スコアリング:

| 観点 | 目標 |
| --- | --- |
| 触った感 | 8/10 |
| 報酬感 | 8/10 |
| 直感性 | 8/10 |
| 長期運用 | 8/10 |
| iPhone実機の軽さ | 8/10 |

合計40/50未満なら、演出を足すのではなく、配置・タイミング・素材品質を見直す。
