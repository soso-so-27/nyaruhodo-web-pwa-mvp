## P0: 公開前に優先して確認/修正すべきもの

### P0-1. 「この子のとっておき」の意味混線

現状:

- `CatsPage.tsx` の写真タブは title が「この子のとっておき」。
- 同セクションの空文言は「ねがおを撮るか、とっておきにのこすと、ここに並びます」。
- 説明文は「毎日のねがおと、選んで残した写真が並びます」。
- 実装上も写真レンズは `ownSleepingPhotos + catGalleryPhotos` の統合表示になっている。

問題:

- 「とっておき」は本来 catGalleryPhotos の100枚枠に見える。
- しかし表示には毎日のねがおも入る。
- ユーザーは「100日でいっぱいになる？」「ねがおもここに保存される？」と迷う。

最小方針:

- セクションタイトルは「この子の写真」または「写真」に戻す方が認知負荷は低い。
- その下に小さく「毎日のねがおと、とっておきが並びます」と説明する。
- 追加ボタンは「+」だけでよいが、アクセシブルラベルは「とっておきを追加」。
- 100枚制限は「とっておき」だけの制限として、追加シート内に閉じ込める。

推奨:

```txt
見出し: この子の写真
補足: 毎日のねがおと、とっておきが並びます。
追加: + (aria-label="とっておきを追加")
```

### P0-2. 基本編集の日付入力のはみ出し/拡大戻り

現状:

- iOSの date input はフォーカス時に拡大/ピッカーUIの癖がある。
- ユーザー報告では「日付の枠がモーダルからはみ出る」「入力後も拡大された感じになる」が発生していた。

問題:

- 入力画面のはみ出しは、操作不能/不安に直結する。
- 認知負荷というより公開前のUXブロッカーに近い。

最小方針:

- 基本編集シートの全 input/select/textarea に `font-size: 16px以上`, `max-width: 100%`, `box-sizing: border-box` を保証。
- 日付は横並びに戻さない。
- `overflow-x: hidden` で隠すだけではなく、固定幅/inline flex の原因をなくす。

### P0-3. ロード画面の出現条件

現状:

- ホームの紙壁ロードは初回表示/リロードのみへ修正済み。
- 内部遷移で都度出る挙動は解消。

残確認:

- `/collection -> /home`, `/cats -> /home`, 設定から戻る、PWA復帰の4パターンで同じか実機確認する。
- もしPWA復帰で毎回出る場合は、初回起動として許容するか、セッション単位にするかを決める。

## P1: 近いうちに直すと満足度が上がるもの

### P1-1. 思い出未開封の印の学習

現状:

- ホームには思い出便到着カードを出さない。
- 未開封の思い出は下部ナビ「うちのこ」の封蝋点と、うちのこ > 記録の今日の1件に出る。

良い点:

- ホーム通知カードの原則「未開封だけ」を守りつつ、ホームの主役を邪魔しない。
- 思い出便が「猫タブに住む」感じは世界観に合う。

リスク:

- 初見では封蝋点の意味が分からない。
- 数字やNEWを使わない方針は良いが、初回だけ意味を学習する機会が必要。

最小案:

- 今日の1件に初めて未開封思い出が出た時だけ、短い補足を出す。

```txt
未開封の思い出には、封蝋の点がつきます。
```

ただし常設しない。2回目以降は消す。

### P1-2. 設定の「保存と復元」名称

現状:

- UIは「いますぐ同期する」1ボタンに集約済み。
- セクション名は「保存と復元」。

問題:

- 操作は「同期」なのに見出しが「保存と復元」なので、ユーザーは「どちらを押すのか」問題からは解放されたが、概念は少し古い。

最小案:

```txt
見出し: アカウント同期
補足: この端末とアカウントの写真をそろえます。
```

ただし、Safari/PWA保存分離の注意は維持する。

### P1-3. オンボーディングの説明量

現状:

- 初回画面に、ねがおを入れるとねこだよりが届くこと、匿名性、SNS外に出ないこと、名前や場所が出ないことをまとめて表示。

良い点:

- 写真共有に関する不安を先に潰している。

リスク:

- 最初の画面で読む量が多い。
- 「まず1枚入れる」心理の邪魔になる可能性がある。

最小案:

- 画面上の本文は2ブロックまでにし、詳細は安全確認シートや「くわしく」に分ける。
- ただし公開前に大きく変える必要はない。写真共有の不安を減らす価値が高い。

### P1-4. 基本表示の読みやすさ

現状:

- 空項目非表示、紙メインに寄せた。
- ユーザーから「読みにくい」「太字や変な枠をやめてほしい」とフィードバックがあった。

見るべき観点:

- 値の列が揃っているか。
- 体重/ワクチンの横並びが狭幅で破綻しないか。
- ラベルと値のコントラスト差が十分か。
- 紙っぽさが情報の読みやすさを邪魔していないか。

最小案:

- ラベル 11-12px、値 15-16px。
- 値は太字にしない。
- 枠で囲わず、薄い区切り線と余白でグルーピングする。
- 日付の補足は「2026年7月2日」だけにし、文章化しない。

## P2: 公開後でよいもの

| 項目 | 理由 |
| --- | --- |
| 完全な用語辞書/ヘルプ | 画面内のコピーが先。ヘルプを増やすと逆に負荷になる。 |
| 文箱の初回チュートリアル | 思い出便の頻度が低いため、実データを見てからでよい。 |
| 設定の再構成 | 現状は機能的に成立。公開前は文言とP0だけでよい。 |
| オンボーディング全再設計 | 写真共有の安全説明が絡むため、実機完走データ後でよい。 |
| 全画面のフォント/余白再統一 | 直す価値はあるが、P0の意味混線を先に直す。 |

## 画面別チェックリスト

### ホーム

- 主役CTAは1つか: はい。撮影前は「ねがおを とる」。
- 副導線が主役を邪魔していないか: 撮影前の小リンク撤去済み。
- 通知カードに開封済みが混ざらないか: ねこだより開封後の「きょうのねこだより」行は撤去済み。
- 思い出便がホーム通知を汚していないか: 到着カード撤去済み。
- 懸念: 思い出未開封の封蝋点の意味は初見では分からない。

### ねこだより/アルバム

- 「おくった」「とどいた」の2面は理解しやすい。
- 写真詳細の削除/通報/非表示は意味が近く、確認文が重要。
- 「ねこだよりに使う」「自分だけ」「アルバムから外す」など、操作の結果が複数あるため、確認文は実データを消すのか表示だけ外すのかを毎回明記する。

### うちのこ 記録

- 「今日の1件」「記念」「足あと」「文箱」「年まとめ」は、意味としては整理されている。
- 今日の1件は表示枠、思い出はコンテンツ種別、記念は事実記録。この分離は維持する。
- 懸念: 今日の1件に出た思い出が文箱にも出ることは許容だが、初見では重複に見える可能性がある。

### うちのこ 写真

- 現在の最大リスク画面。
- 「この子のとっておき」という見出しと、実際に「毎日のねがお + とっておき」が並ぶ仕様が合っていない。
- 推奨は「この子の写真」へ戻し、追加ボタンだけ「とっておき」を維持すること。

### うちのこ 基本

- 表示側は「入力していないものを見せない」方針でよい。
- 編集側は説明文を削った方向でよい。
- 日付/保存バー/横幅の破綻は実機で継続確認。

### 設定

- アカウント削除導線、おしらせ、思い出停止、同期集約はよい。
- 設定は日常画面よりも明快さ優先。ひらがな声を過度に入れない。
- セクション名「保存と復元」は「アカウント同期」にするとさらに低負荷。

### オンボーディング

- 不安解消の情報は必要。
- ただし最初の画面の本文量はやや多い。
- 公開前は大改修しない。安全説明の分割はP1。

## 判断

現在のアプリは、ホームとねこだよりの主導線はかなり整理されている。一方で、最近の改善で写真ドメインが豊かになった分、`ねがお` / `とっておき` / `この子の写真` の言葉が再び少し混ざっている。

公開前の認知負荷対策としては、演出や画面数を増やすより、まず「写真タブの名前」と「基本編集の実機破綻」を直すのが最も効果が高い。
