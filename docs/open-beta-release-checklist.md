# ねてるねこ 告知前チェックリスト

告知してよい状態かを確認するための最小チェック。
毎回、`main` 反映後の本番URLで確認する。

## 1. 本番DB

- [ ] `npm run check:release` が通る
- [ ] `supabase migration list` で Local / Remote がすべて一致している
- [ ] `cat_moments` が本番RESTで見える
- [ ] `cat_moment_deliveries` が本番RESTで見える
- [ ] `cat-photos` Storage にアクセスできる

## 2. iPhone PWA

- [ ] Safariで本番URLを開く
- [ ] ホーム画面に追加する
- [ ] PWAから起動する
- [ ] Googleアカウントで接続する
- [ ] PWAを閉じて再起動してもログインが残る

## 3. 保存

- [ ] 寝顔を1枚入れる
- [ ] `とったねがお` の数が増える
- [ ] 必要なら `とどかないようにする` を切り替えられる
- [ ] 設定で保存を実行する
- [ ] 保存結果に `とったねがお` の件数が出る
- [ ] エラーが出ていない

## 4. 復元

- [ ] 別PWA、または新しく追加したPWAを開く
- [ ] 同じGoogleアカウントで接続する
- [ ] アカウントから復元する
- [ ] 猫が戻る
- [ ] `とったねがお` が戻る
- [ ] `とどいたねがお` が戻る
- [ ] ボックスで写真が表示される
- [ ] 復元結果に件数が出る
- [ ] エラーが出ていない

## 5. 写真体験

- [ ] 寝顔を入れる導線が最初に分かる
- [ ] 写真選択後の確認画面で縦写真が崩れない
- [ ] 写真選択後の確認画面で横写真が崩れない
- [ ] `自分だけ` と `届く` の意味が誤解されない
- [ ] 受け取った寝顔を `とっておく` できる
- [ ] 受け取った寝顔を閉じられる
- [ ] 通報は受け取り時だけに出ている

## 6. 設定と安心

- [ ] 保存/復元が何をするか分かる
- [ ] 失敗時に原因が少し分かる
- [ ] 写真の保存場所の説明が読める
- [ ] プライバシーポリシーへ行ける
- [ ] 利用規約へ行ける
- [ ] データ削除導線が見える
- [ ] 問い合わせ先を告知文またはプロフィールに置く

## 7. 告知文

- [ ] 1文で体験が分かる
- [ ] 長い説明を読ませない
- [ ] `うちの寝顔を1枚おくと、どこかの寝顔が1枚届きます。` を基本にする
- [ ] β版であることを添える
- [ ] 不具合報告先を添える

## NG

- [ ] 写真が復元されない
- [ ] 猫だけ復元されて成功に見える
- [ ] 保存/復元のエラーが何も表示されない
- [ ] ログインが再起動で消える
- [ ] 本番DB migration が未適用
- [ ] 画面に文字化けがある
