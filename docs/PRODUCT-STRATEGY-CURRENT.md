# ねてるねこ 現行プロダクト方針

更新日: 2026-07-23
状態: 現在の判断基準

この文書は、現在の事業目的、プロダクトの役割、未決事項、実行順を一か所に固定する。
課金内容と実行順については、`docs/neteruneko-business-strategy-v1.2.md` と
旧 `docs/ROADMAP.md` より本書を優先する。
語彙、無料原本、運用、プライバシー等の恒久ルールは `AGENTS.md` を優先する。

## 1. 事業目的

- 自分で顧客接点、猫の記録、継続課金を持つプラットフォームを作る
- 労働時間の切り売りではなく、デジタル月額収益を積み上げる
- 最初の事業領域は、自分の飼育経験と既存発信を生かせる猫toCとする
- 長期目標は月商500万円だが、現在の実行目標は最初の継続利用と最初の10課金世帯である

## 2. 今作るプロダクト

### 対象

猫と暮らし、写真は撮るものの、整理や記録を新しい作業としては続けたくない人。

### 表向きの約束

> ほかの猫との交換を楽しみながら、うちの猫の写真が自然に残っていく。

### 仕組み

1. うちの猫のねがおを一枚残す
2. よる8時に、ほかの家の猫が4匹とどく
3. 一匹を選んで、ねこだよりとして残せる
4. 同時に、うちの猫の写真が猫ごとの記録として育つ

### 役割の分離

| 層 | 役割 |
|---|---|
| うちの猫 | プロダクトの中心。本人の写真と時間の蓄積 |
| 4匹交換 | 翌日も一枚残したくなる楽しさ |
| ねこだより | 選んだ他家の猫との小さな履歴 |
| 有料プラン | 無料利用後に生まれる不足を埋める。内容は未決 |
| 物販 | 蓄積から必要な時に買える別商品。補助収益 |

寝顔は現在の体験を小さく成立させるための制約であり、事業目的そのものではない。
対象を広げる場合も、うちの猫の記録と4匹交換を弱める「何でも猫アプリ」にはしない。

## 3. 作らないもの

- 入力項目の多い健康・タスク管理
- 健康状態や性格の診断
- 公開フィード、フォロワー、ランキング等のSNS
- 病院、葬祭、入居、保護団体を同時に扱う業務プラットフォーム
- 月額理由が未確認のまま作るアルバム、AI分析、家族共有の全部入り

## 4. 無料と有料

### 無料として固定

- 一枚残して4匹から一匹選ぶ日々の交換
- 自分で残した原本
- 選んで残したねこだより
- 現在提供している猫別写真一覧と年表示

### 有料について固定

- デジタル月額課金を事業の中心として検証する
- 検証価格は月980円（税込）／世帯
- 原本を人質にしない
- 物販は別料金であり、デジタル課金の代わりにしない

### 有料について未決

- 誰が払うか
- 無料で満たされない何を解決するか
- 多頭無制限、家族共有、交換の深掘り、自動発見、デジタル冊子のどれを含めるか
- どの機能の組み合わせなら毎月払い続ける理由になるか

月アルバム案と非アルバム3案は、いずれも内部仮説であり商品決定ではない。

## 5. 現在地

| 領域 | 状態 |
|---|---|
| 4匹から一匹選ぶ | 本番で成立。従来より楽しいという定性反応あり |
| オンボーディング | 4匹選択を体験できる |
| うちの猫 | 写真一覧・年表示等はあるが、主役としての情報設計は途中 |
| ねこだより | 選ぶ楽しさは改善。翌日・1週間の効果は未確定 |
| 有料プラン | 対象者・価値とも未確定 |
| 月アルバム試作 | 内部仮説 |
| 非アルバム3案試作 | 内部仮説比較。ユーザー向け商品画面ではない |
| 本番課金 | 未実装。公開承認なし |

## 6. 今の検証方法

現在の利用人数では、継続率の数値だけで良否を判定しない。
母数が増えるまで改善を止めるのではなく、利用者一人ずつの行動と、
説明なしで操作した時の理解を見て、構造仮説を小さく直す。

| 仮説 | 一人ずつ見ること |
|---|---|
| 初回成立 | 一枚を残す → 4匹をひらく → 一匹を選ぶ、を説明なしで完了できたか |
| 役割理解 | 自分の写真は「うちのこ」、選んだ他家の猫は「ねこだより」にあると分かったか |
| 完了理解 | 一匹を選んだ後、今日の操作が終わったと分かったか |
| 再訪理由 | 次にいつ何が起きるかを、画面から理解できたか |
| 蓄積価値 | 「うちのこ」で、自分の写真が残った事実を確認したか |
| 課金価値 | 一つの具体的体験を使った後、月980円で開始すると明言したか |

集計率は方向を見る補助値として残すが、少人数の段階では一人の途中離脱や
誤解を優先して直す。表示数、アンケート上の「良さそう」、3案の人気だけでは
課金根拠にしない。

## 7. 実行順

1. 正典と古い事業文書の矛盾を解消する
2. 下部ナビで「うちのこ」を中心にし、プロダクトの主役を画面構造へ反映する
3. 「うちのこ」と「ねこだより」の重複をなくし、開いた時の役割を明確にする
4. 新しい入力を増やさず、保存先・完了・次回を具体的に伝える
5. 集計率を待たず、一人ずつの導線と少人数の操作観察で誤解を直す
6. 同時に集客を続け、母数が増えた時点で翌日・初週の集計判断を加える
7. 繰り返した人の行動から、有料対象者を一種類に絞る
8. 料金表ではなく、一つの有料価値を実際に使う試作を作る
9. 価値への反応と月980円への反応を別々に確認する
10. 最初の10課金世帯と2か月目の継続を確認してから獲得を広げる

無料導線が続かない場合は有料機能を増やさず、4匹交換とうちの猫の接続を直す。

## 8. 保留する案

- 入居パスポート
- 病院連携
- 葬祭追跡
- トリミング承認
- トレーナー宿題
- MCP、業務SaaS、海外向け別アプリ
- 物販中心への転換

これらは捨てないが、現在の猫toCで継続と最初の課金を確認するまで実装しない。
