# MVP v0.2.3 現状整理

## 1. MVP v0.2.3 の位置づけ

にゃるほどは、猫の正解を当てるアプリではない。
飼い主の迷いを減らし、猫との暮らしを自然に残すアプリである。

現在は、単なる診断・記録アプリではなく、以下の方向へ寄せている。

猫を見ていたら、暮らしがたまっていくアプリ

MVP v0.2.3 は、DB本格化・認証・家族共有前の localStorage MVP 段階にある。
猫プロフィールや active cat は localStorage を中心に扱い、既存の `events` などは Supabase に保存する。
ただし、`cats` テーブル作成や認証前提のDB化はまだ行わない。

## 2. 3タブ構造

現在のタブは以下の3つ。

- 今日
- コレクション
- ねこ

それぞれの役割は以下。

- 今日: 見たままを残す / 入力・観察
- コレクション: たまったものを見る / 写真・記録のコレクション棚
- ねこ: 猫プロフィール・管理

今日は入力する場所、コレクションは残ったものを見返す場所、ねこは猫の情報を整える場所として分ける。

## 3. /home の現在仕様

`/home` は、今日の猫をのぞき、見たままを残す場所である。

現在の主な要素:

- 猫カード
- さいきんの{猫名}
- いまの{猫名}
- {猫名}の1日ミニマップ
- いつもの様子
- ちょっと気になる

入力UIは、以下を1つの入力カード内に統合済み。

- いつもの様子
- ちょっと気になる

現在の保存挙動:

- `いつもの様子` は `current_state` として `events` に保存する
- `ちょっと気になる` は `concern` として `events` に保存する
- `ちょっと気になる` 押下後は `/diagnose` に遷移しない
- `/diagnose` ページと診断ロジックは削除せず、将来の詳しい相談導線として残す

保存成功時:

```text
✓ 残しました
```

active cat にとって初めての行動ログ項目に対応する記録だった場合:

```text
「◯◯」の記録が入りました
```

ここでは「みつけました」は使わない。
写真つきのポーズ登録が入るまでは、今日タブのワンクリック記録は「行動・状態の記録が入った」程度に留める。

保存失敗時:

```text
保存できませんでした。
通信状態を確認して、もう一度お試しください。
```

このエラーは入力カード内に表示し、成功表示とは混ぜない。

## 4. /collection の現在仕様

`/collection` は、たまったものを見る場所である。
将来的には写真・ポーズ図鑑が主役になる「コレクション」として扱う。

現時点では写真機能が未実装のため、既存 `events` 由来の表示はポーズコレクションの発見判定には使わず、補助的な「最近見た」「このごろ」として扱う。

上部説明文:

```text
写真や記録が、少しずつここに残ります。
```

### 最近見た

役割:

- 最近の記録から見えたものを表示する
- active cat の直近 `events` から最大3件を表示する
- 写真がない段階では、写真カードではなく軽いチップ/小カードとして扱う

空状態:

```text
まだこれから
写真を残すと、ここに並びます。
```

写真追加導線はまだ作らない。

### ポーズコレクション

役割:

- 将来の写真・ポーズ図鑑の骨格
- 今日タブの行動ログとは別の、写真で集めたくなる猫ポーズカテゴリを表示する
- 写真つき登録が入るまでは、各項目は `まだこれから` と表示する
- 空カウント時は `写真はまだこれから` と表示する

方針:

- 達成率は出さない
- コンプリート圧は出さない
- 未発見をロックや失敗に見せない
- 既存 `events` 由来の `ねむる` / `毛づくろい` / `ごはん` などを `記録あり` 判定に使わない
- 「みつけた」は、写真つき登録や手動ポーズ追加が入った段階で強く使う

初期の写真ポーズカテゴリ:

- へそ天
- 箱入り
- のびー
- ごめん寝
- 香箱
- 窓辺監視
- 液体化
- まるまり
- しっぽピーン
- おすわり
- 変な寝相
- ふみふみ

### このごろ

役割:

- active cat の直近 `events` から軽いふりかえりを表示する
- 分析ではなく、短いメモとして扱う
- スコア、グラフ、達成率は出さない

文言は pose の tone に応じて出し分ける。

- normal: よく見かけます / 少し多めです
- social: 見えてきました
- concern: 残っています

concern 系でも不安や医療診断に見せず、日常の記録としてやわらかく扱う。

## 5. 行動ログ対応と tone 分類

signal から行動ログ表示への対応は `src/lib/collection/poses.ts` に集約する。

signal -> 行動ログ表示:

- `sleeping` -> ねむる
- `grooming` -> 毛づくろい
- `playing` -> あそぶ
- `food` / `after_food` / `eating` -> ごはん
- `toilet` -> トイレ
- `purring` -> ごきげん
- `meowing` -> おしゃべり
- `following` -> ついてくる
- `restless` -> そわそわ
- `low_energy` -> 休む
- `fighting` -> ケンカ
- `unknown` -> よくわからない

tone 分類:

normal:

- ねむる
- 毛づくろい
- あそぶ
- ごはん
- トイレ
- ごきげん

social:

- おしゃべり
- ついてくる

concern:

- そわそわ
- 休む
- ケンカ
- よくわからない

文言方針:

- normal: `よく見かけます` / `少し多めです`
- social: `見えてきました`
- concern: `残っています`

social は不安扱いしない。
concern は「多い」「傾向」ではなく、記録として残っている程度に留める。

## 6. /cats の現在仕様

`/cats` は、猫プロフィール・管理の場所である。

## 8. UX方針

MVP v0.2.3 のUX方針:

- 記録を義務にしない
- 達成率を出さない
- コンプリート圧を出さない
- 収集ゲームにしすぎない
- 写真機能が入るまでは、events 由来の表示は `最近見た` / `このごろ` として弱く扱う
- ポーズコレクションは写真・ポーズ図鑑の骨格として置き、行動ログ由来の `記録あり` 判定には使わない
- 写真つき登録が入った段階で `みつけた` を強く使う
- 医療っぽくしすぎない
- 不安を煽らない
- 断定しすぎない
- ホームを重くしすぎない
- コレクションは分析画面ではなく、写真や記録が少しずつ残る棚として見せる

今日タブでは、まず見たままを残せることを優先する。
コレクションでは、残したものや将来の写真が少しずつ返ってくる感覚を優先する。

## 11. デザイン方針

MVP v0.2.3 以降のUIは、E方向「猫が主役でUIが脇役」を基本にする。

ベースカラーは白〜グレージュとし、茶色や濃いオレンジを主役にしない。写真や猫イラストが最も映えるように、UIは静かで、上品で、長く使える方向へ寄せる。

採用する方向:

- 猫写真・猫の姿を主役にする
- UIは控えめにする
- グレージュ、アイボリー、薄いグレーを中心にする
- 余白を広めに取る
- カードは軽く、影や線は控えめにする
- コレクションは写真が入った時に映える棚として扱う

避ける方向:

- 強い茶色
- 濃いオレンジ
- ゲームっぽすぎる演出
- 絵本、紙、スタンプ風への寄せすぎ
- カラフルすぎるタイル
- 過剰な装飾

この方針は、写真アップロードや写真ポーズコレクションを将来入れる前提の土台として扱う。今すぐ全面的な色変更やUI改修は行わない。
