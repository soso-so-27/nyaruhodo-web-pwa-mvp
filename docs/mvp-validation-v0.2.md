# MVP検証版 v0.2

## 目的

猫の正解を当てるのではなく、飼い主の迷いを減らす体験が成立しているかを見る。

MVP v0.2 では、初回オンボーディング、日常記録、気になる行動の診断、診断結果後の行動決定までが、ひとつの流れとして自然に使えるかを確認する。

## 現在のコア体験

1. `うちの猫、どんなタイプ？` で入る
2. Q1〜Q15でこの子の傾向を少し知る
3. ホームで今日の猫を見る
4. 見たままをひとつ残す
5. 気になるときは診断する
6. 診断結果から次の行動を決める
7. 違うかもも記録できる

## 現在できること

- 初回診断オンボーディング
- Q1〜Q15
- `cat_profiles` 保存
- `active_cat_id` 保存
- `understanding` 反映
- ホーム表示
- 日常記録
- 気になる行動診断
- `feedbacks` 保存
- `hint_feedbacks` 保存
- `/cats` で猫追加/名前変更
- オンボーディング `typeKey` による `/diagnose` 弱補正

## まだやらないこと

- `/record`
- `/review`

- `cats` テーブル
- 家族共有

## 検証シナリオ

1. 初回オンボーディング
2. 3問だけ答えてホーム
3. 15問まで答えてホーム
4. 日常記録
5. 気になる行動診断
6. 違うかも
7. 多頭飼い切り替え
8. `/cats` で猫管理

## 評価観点

- やりたくなるか
- 何をすればいいか分かるか
- 決めつけ感がないか
- 不安を煽らないか
- 押した後に迷わないか
- また開きたくなるか
- 記録が義務に感じないか
- 猫ごとの文脈が混ざらないか
- オンボーディング回答が診断に活きているか

## 次の改善判断基準

### 高優先度

- 初回で離脱しそう
- ホームで何をすればいいか分からない
- 診断結果が決めつけに見える
- 不安を煽る
- 猫ごとの文脈が混ざる
- オンボーディング回答が診断に活きている感じがない

### 中優先度

- 文言が少し硬い
- UIが少し重い
- 報酬感が弱い
- 猫管理ページが薄い

### 低優先度

- Q16〜Q30
- `/record`
- `/review`
- 通知
- 認証
- DB化

## 検証時の注意

- オンボーディング結果はタイプ確定ではなく、最初の手がかりとして見る
- `/diagnose` のオンボーディング補正は弱く扱い、今の行動を主軸にする
- `feedbacks` と `hint_feedbacks` は別用途として確認する
- localStorage ベースの多頭飼い分離であり、端末間同期はまだ対象外とする
- DB/RLSは v0.2 では変更しない
