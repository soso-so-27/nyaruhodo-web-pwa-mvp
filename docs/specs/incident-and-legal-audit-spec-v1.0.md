## タスク① 本番データ1337件削除の経緯調査（P0・フォレンジック）

### 背景

2026-07-04時点の状態報告に「pending旧データ削除の記録: **1337件削除、残り0件**」とあるが、
この削除はレビュー側と合意した工程（管理キューで12件を目視→approve/reject消化）と異なる。
削除の実体・経路・範囲を確定し、記録として残す。**責任追及ではなく、統制の確認と記録が目的。**

### 調査項目

1. **何が消えたか**
   - 削除対象はどのテーブルか（cat_moments? cat_moment_deliveries? 両方?）
   - 1337件の内訳: 以前確認した「pendingのままdelivered」16件（moment 12件）は含まれるか
   - 削除条件（WHERE句相当）は何だったか
2. **誰が・何で・いつ消したか**
   - 実行経路: SQL直叩き / 管理API / cleanupスクリプト / migration のどれか
   - リポジトリ内を検索: cleanup系スクリプト・API route・直近コミットのdiffに
     削除処理が含まれていないか（`git log --all -p -S "delete" --since=2026-07-03` 等で特定）
   - 実行時刻の痕跡（Supabaseログ、app_events、コミット時刻）
3. **削除前の目視工程は踏まれたか**
   - 12件の元写真を目視した記録・成果物はあるか
4. **復元可能性**
   - 削除実行時刻より前のPITR/バックアップが存在するか（Supabaseダッシュボードで確認）
   - 復元可能な期限（PITRの保持期間）を明記
5. **副作用の確認（SQLは読み取りのみ）**
   ```sql
   -- 孤児deliveryの有無（momentが消えてdeliveryだけ残っていないか）
   select count(*) from cat_moment_deliveries d
   left join cat_moments m on m.id = d.cat_moment_id
   where m.id is null;

   -- 現在の全量スナップショット
   select moderation_status, count(*) from cat_moments group by 1;
   select status, count(*) from cat_moment_deliveries group by 1;
   ```
   - Storage側の対応オブジェクト（写真ファイル）は削除されたか、孤児として残っているか

## タスク② 法務文書の棚卸（P0・読み取りと報告のみ）

### 背景

利用規約・プライバシーポリシー・削除案内は作成済みとのこと。
先行ユーザー受け入れ（投稿3）前の最終確認として、所在・網羅性・導線を棚卸する。
**文面の書き換えは行わない**（内容の最終判断は中西、文面レビューは別途）。

### 調査項目

1. **所在の確定**
   - 各文書のファイルパス・ルーティングURL・最終更新日
   - 対象: 利用規約 / プライバシーポリシー / 削除案内（アカウント・写真）/
     特定商取引法に基づく表記（課金があるため必須）/ 問い合わせ先
2. **網羅性チェック（各文書に該当記述が「ある/ない」の表を作る。内容の質は評価しない）**

   利用規約:
   - [ ] 写真の取り扱い: ユーザーが権利を保持するか、運営への利用許諾の範囲
     （他ユーザーへの配達・モデレーション目視・サムネイル生成に必要な最小許諾）
   - [ ] 写真の削除: ユーザーが削除した場合、配達済み相手側での扱い
   - [ ] 禁止事項（猫以外・不適切写真・他者の写真の無断投稿）
   - [ ] 事前審査（モデレーション）の存在と、reject時の扱い
   - [ ] 課金: 月額・自動更新・解約方法・返金方針
   - [ ] サービスの変更・終了時の扱い
   - [ ] 準拠法・管轄

   プライバシーポリシー:
   - [ ] 取得する情報の列挙（写真・匿名ID・メール等の認証情報・app_events相当の行動ログ）
   - [ ] 利用目的
   - [ ] 第三者提供の有無（配達の仕組み上、写真が他ユーザーに表示されることの明記）
   - [ ] 外部サービスの列挙（Supabase / Vercel / Stripe / Google認証）
   - [ ] 保存期間・削除請求への対応方法
   - [ ] 問い合わせ窓口

   削除案内:
   - [ ] 写真1枚単位の削除方法
   - [ ] アカウント削除（退会）の方法と、削除されるデータの範囲
   - [ ] 配達済み写真の相手側での扱いの説明

   特定商取引法に基づく表記:
   - [ ] 事業者名・所在地・連絡先・販売価格・支払時期/方法・解約条件

3. **導線チェック（実装の確認。修正はしない）**
   - 設定から各文書まで何タップか（目標: 削除案内まで2タップ以内）
   - オンボーディング/課金導線に規約同意のステップがあるか
   - 未ログイン状態でも規約・プラポリが読めるか（ストア審査・外部確認用）
