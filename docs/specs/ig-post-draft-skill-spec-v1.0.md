# skill仕様: ig-post-draft（投稿ドラフト生成）v1.0

> 対象: Codex実装用。自己完結。
> 前提: `docs/marketing/MARKETING-CANON.md` v0.1 がリポジトリにコミット済みであること。
> 本skillは MARKETING-CANON §8「委任する（定型）」の1番と2番を実装する。

---

## 1. 目的

中西が「写真の説明＋一言メモ」を渡すと、MARKETING-CANONに準拠した
IGフィード投稿のキャプション案を2種と、規範検査の結果を返す。

**本skillは案を作るのみ。投稿の可否・最終文言の決定は行わない。**
出力の冒頭に必ず「以下はドラフトです。最終判断は中西さんです。」を含める。

## 2. 配置

```
.codex/skills/ig-post-draft/
├── SKILL.md          # 本仕様の運用版（トリガー条件・手順）
├── canon-check.md    # 検査ルール（§5から生成、CANON §3のミラー）
└── examples.md       # 良い例・悪い例（§7）
```

- CANON本体は `docs/marketing/MARKETING-CANON.md` を単一の真実とし、
  `canon-check.md` はそこから機械的に抽出したルール表。
  **CANON改訂時は canon-check.md の再生成を必須とする**（乖離が最大のリスク）。

## 3. 入力

ユーザー（中西）からの自然文。最低限、以下の2要素を含む:

1. **写真の説明** — どんなねがおか（例: ソファの背もたれで仰向け、前足が浮いている）
2. **一言メモ** — 伝えたい断片（例: 最近ここがお気に入り / 開発でボタンの角丸を3日悩んだ）

任意要素:
- 投稿の種別指定: `むぎ日常` / `build in public`（未指定時はメモ内容から推定し、推定結果を明示）
- 現在の告知フェーズ: P1/P2/P3（未指定時は **P1 とみなす**。CANON §5参照）

## 4. 出力フォーマット（固定）

```
【ドラフト】以下はドラフトです。最終判断は中西さんです。

■ 種別: むぎ日常 / build in public（推定の場合はその旨）
■ フェーズ: P1（このフェーズで出せないもの: アプリ名・リリース日）

── 案A（トーン: 打ち明け）
<キャプション本文>

── 案B（トーン: 情景から入る）
<キャプション本文>

■ 規範検査
- 禁止表現: 検出なし / 検出あり（該当箇所と条項）
- 語彙ロック: 準拠 / 違反（該当箇所）
- フェーズ制約: 準拠 / 違反（P1でアプリ名が出ている 等）
- 絵文字: なし / あり（位置)

■ 補足（任意）
写真とキャプションの整合で気になる点があれば1点まで
```

## 5. 検査ルール（canon-check.md の内容）

CANON §3・§2・§5 から抽出。出力前に自己検査し、違反があれば
**その案を破棄して書き直す**（違反を含んだまま提示しない）。
書き直しても解消できない場合のみ、違反箇所を明示して提示する。

### 5.1 禁止語リスト（完全一致＋活用形）
- 死関連: 一生, 最期, 見送る, 見送り, 虹の橋, 天国
- 煽り: 今だけ, お早めに, 残りわずか, 見逃すな, 必見, バズ, 緊急
- 誇示: 突破, ランキング, 話題の, 大人気, 急上昇
- 依頼: 拡散, シェアお願い, フォローお願い, いいねお願い
- ゲーミフィケーション: 連続記録, ストリーク, ポイント, ランク, 達成
- リファラル報酬: 紹介で, 招待特典

### 5.2 語彙ロック（アプリ内動作に言及する場合のみ適用）
- 寝顔写真→ねがお / 通知・配信→ねこだより・とどく / 開封→ひらく
- 「毎日」→「一日一枚」を推奨（強制ではなく指摘レベル）

### 5.3 フェーズ制約
- P1: アプリ名「ねてるねこ」・リリース日・ストアリンクを含まない
- P2: リリース日・ストアリンクを含まない
- P3: 制約なし（ただし数字誇示は恒久禁止）

### 5.4 形式
- 絵文字: 検出したら違反
- ハッシュタグ: 3個以内、かつ煽り系タグ（#拡散希望 等)は禁止
- 文字数: 400字以内目安（超過は指摘レベル、違反ではない）

## 6. キャプション生成の文体規則

- 文体: です・ます調の打ち明け。主語は「私」または無主語。企業文体禁止
- 構成: 写真の情景 → メモの断片 → （必要なら）静かな締め。オチや教訓をつけない
- 案Aと案Bは**書き出しの視点を変える**（A: 近況・時間から / B: 情景・描写から）。
  トーン自体は両案とも静けさの範囲内であること
- リズム: 短文基調。体言止めの多用禁止（広告文体になるため）

## 7. examples.md に含める実例

### 良い例（このスレッドで確定したトーン）
> ひさしぶりの投稿です。むぎは元気で、あいかわらずよく寝ています。
> ストーリーズで気づいた方もいるかもしれませんが、いま、この寝顔のためにアプリをひとつ作っています。（後略）

### 悪い例1（煽り・誇示）
> ついに完成間近！🐱 むぎのアプリ、絶対見逃さないで！フォロワーのみんなに最速でお知らせします✨
→ 違反: 絵文字, 煽り（見逃さないで）, 依頼文体, 「！」の連打

### 悪い例2（企業文体）
> 【お知らせ】この度、新サービスをリリースする運びとなりました。
→ 違反: 宣言体, 主語が企業, 打ち明けの文体でない

## 8. 非対象（このskillがやらないこと）

- 投稿タイミングの提案・スケジューリング
- ストーリーズ用文面（フィード専用。ストーリーズはラフでよいため規範化しない）
- 画像加工・写真選定（選定基準はCANON外の判断領域）
- DM・コメント返信の文案
- 週次レポート（別skill: ig-weekly-report として将来定義）

## 9. 受け入れ確認（中西によるテスト）

1. `むぎ日常` の入力1件 → 2案とも検査全項目「準拠」で出力される
2. わざと「一生」を含むメモを入力 → 出力に「一生」が含まれず、
   検査欄に「入力メモに禁止語が含まれていたため言い換えた」旨が明示される
3. P1指定でアプリ名を含むメモを入力 → 出力にアプリ名が含まれない
4. 出力冒頭に必ずドラフト宣言がある
