## 1. 調査範囲

ホーム（きょう）の20時前後の状態遷移と、ひらく〜表示までの一連:

1. **20時をまたぐ瞬間**: 待ち状態（封筒＋「もうすぐ、とどく」）から到着状態への切り替わりは
   何がトリガーか（ポーリング? 再訪時判定? リアルタイム?）。画面を開いたまま20時を迎えた場合と、
   20時以降に開いた場合で見え方が違うか
2. **ひらく操作**: 現在の操作は何か（長押し? タップ?）。P0バグ修正後の実装
   （pointer events / setPointerCapture / 写真レイヤーの遅延マウント）の現状
3. **ひらいた後の表示**: StampPair（自分のねがお大＋受信切手が右上に傾き）への遷移。
   アニメーションの有無・種類・所要時間
4. **モーションの残骸**: 試みて取りやめたモーション実装の痕跡
   - 未使用のCSS animation / transition / keyframes
   - フラグで無効化されたコード・コメントアウト・dead code
   - 関連コミット（`git log --oneline -- <該当ファイル>` で該当期間を特定し、
     試行→取りやめの経緯をコミット単位で3〜5行に要約）
5. **spec乖離**: `docs/specs/` 内の開封・モーション関連spec（封筒静置＋時刻グラデーション、
   StampPair、開封セレモニー等）と現実装の差分。「specにあるが実装にない」
   「実装にあるがspecにない」を表で
