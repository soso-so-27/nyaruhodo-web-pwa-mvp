# ねてるねこ UX改善 spec v1.0

対象: タブ切替の遅さ、画像読み込みの遅さ、文字コントラスト、カメラボタンのアフォーダンス。

対象外: ホーム state1 の手紙視認性、アルバム情報設計、猫ページ情報設計、アイコン素材。

## 1. 画像パイプライン

### 1.1 サムネイル生成

アップロード時に以下の3サイズを保存する。

| 用途 | サイズ | 形式 |
| --- | --- | --- |
| thumb | 長辺 400px | WebP, quality 70 |
| display | 長辺 1200px | WebP, quality 80 |
| original | 原寸 | 受領形式のまま |

Storageの変換機能が使える場合はそれを優先する。使えない場合のみアップロードAPIまたはクライアント側生成で補う。

既存写真のバックフィルスクリプトを用意する。冪等であること。

### 1.2 配信ルール

- アルバム一覧、ねこタブ、封筒ミニチュアなどの一覧系は thumb のみを使う。
- 開封ビュー、写真詳細は display を使う。
- original は表示用途に使わない。
- `next/image` を使い、`sizes` を指定する。

### 1.3 現像プレースホルダー

読み込み中はスピナーではなくブラーから鮮明へ変化させる。開封フローの現像アニメーションと同じ言語に揃える。

## 2. タブ切替の体感速度

- タブ切替でページコンポーネントをアンマウントしない。
- 表示切替は CSS または React Activity などで状態とスクロール位置を保持する。
- 一度取得したデータはクライアントキャッシュに保持し、再訪時は即表示して裏で再検証する。
- 初回ロード時に隣接タブを低優先度で先読みする。

受入基準: 2回目以降のタブ切替で前回内容が100ms以内に表示され、白画面やスピナーが出ない。

## 3. 文字コントラスト

テキスト色は `src/app/tokens.css` の実在トークンに統一する。

- `--ink`: 本文
- `--ink-soft`: 補助、キャプション
- `--ink-faint`: 非活性のみ

（2026-07-07改訂: 旧記載の `text-primary` / `text-secondary` / `text-disabled` は
実装に存在しない名称だったため実名へ修正）

`--ink` と `--ink-soft` は背景に対して 4.5:1 以上。24px以上の見出しのみ 3:1 まで許容する。

## 4. カメラボタンのアフォーダンス

- 紙系の面（`--paper` 系）＋細い線＋`--shadow-e1` で押せる見た目にする。
  有彩色の塗りは使わない。塗りボタンは1画面1つまで（DESIGN-CANON §5-3準拠。
  2026-07-07改訂: 旧「塗りと影」の指示をデザイン正典に整合させた）。
- アイコンのみの場合は「とる」などのラベルを併記する。
- タップ領域は最小 44 x 44px。
- 押下時は scale 0.96 と影の変化を100ms程度で返す。
- ホーム state1 から state4 まで成立させる。

## 5. 計測

- 画像読み込み完了時間。
- タブ切替から表示完了までの時間。
- 既存の `trackProductEvent()` の命名規則に従う。

## 6. 検収

- Playwright: タブ切替でスクロール位置と選択状態が保持される。
- 実機: アルバム一覧のスクロール時に画像が順次ブラーから鮮明になる。
- Fast 3G相当で一覧表示を確認する。
