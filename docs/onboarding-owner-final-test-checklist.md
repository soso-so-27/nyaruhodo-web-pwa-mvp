# オンボーディング 最終実機チェックリスト

更新日: 2026-07-21
対象: 投稿3の公開前に、オーナーが実機で確認する最小関門と回帰確認

## 0. 準備

- 本番入口: `https://neteruneko.jp/onboarding?src=instagram_bio`
- テスト用reset入口: `https://neteruneko.jp/onboarding?src=instagram_bio&reset_onboarding=1`
- reset入口は、その端末・そのドメインのオンボ用ローカル状態だけを消す。サーバ上のアカウントや写真は削除しない。
- PWA追加案内はシークレットタブで制限される場合がある。案内確認だけは通常タブで行う。
- 問題が出たら、端末、OS、開いたブラウザ、時刻、画面を記録する。

## 1. 投稿3の公開関門

投稿当日はA〜Cを行う。深い回帰は§2へ分け、毎回の公開判断を重くしすぎない。

### A. Instagram内ブラウザ

1. Instagramアプリ内で本番入口を開く。
2. 「SafariやChromeで開く」案内が出ることを確認する。
3. 「URLをコピーする」で完全なURLをコピーできることを確認する。
4. 「このまま試す」から、写真選択と即時便の開封まで進める。

合格: 一度目のJPEG/PNG選択で進み、外部ブラウザへ移る方法と、そのまま試す方法の両方が使える。

### B. まっさらなAndroid Chrome

1. reset入口を開く。
2. 写真を1枚入れる。
3. その場で届いた封筒をひらき、写真が表示されることを確認する。
4. 「ねてるねこを はじめる」から猫の名前を入れ、ホームへ進む。
5. 2枚目を求める案内が出ず、最初の写真が `次の20時便` に予約済みとして扱われることを確認する。
6. 20時前に開始した場合は当日、20時以降に開始した場合は翌日の20時便が対象になることを確認する。
7. 保存失敗の警告が出ず、予約済みの別写真がある場合は上書きされないことを確認する。
8. 通常タブでAndroid向けのホーム画面追加案内を確認する。

合格: 1枚の写真で初回の即時便と次の20時便予約が成立し、2枚目の要求、`QuotaExceededError`、成功表示だけの偽陽性がない。

### C. iPhone Safari / PWAスポット確認

1. Safariの通常タブで本番入口を開く。
2. 写真選択が一度目から開き、初回の即時便まで進めることを確認する。
3. iPhone向けの共有ボタンと「ホーム画面に追加」案内を確認する。
4. 既存PWAを完全終了して起動し、ホームが開くことを確認する。

合格: SafariとPWAのどちらも操作不能にならず、写真や猫情報が欠けない。

## 2. 変更時に行う回帰確認

次のシナリオは毎日の投稿関門ではない。該当コードを変更したとき、または週次の回帰で行う。

### D. 中断と再訪

写真投入後、封筒の前、開封後、ホーム到達後で一度閉じ、同じURLから再訪する。端末間引き継ぎも確認する。

合格: 意味のある直前状態から続き、写真やねこだよりが二重にならない。予約が欠けている場合だけ対象便の翌日05:00までは補完され、既存の別写真は上書きされない。05:00以降の古い初回写真は新しく予約されない。

### E. 20時境界

20時直前と20時直後にreset入口から初回写真を入れる。20時前は当日、20時以降は翌日の便を対象にし、どちらも画面上は `次の20時便` と表示されることを確認する。

合格: オンボの即時便と通常20時便の説明、日付、保存先が混ざらない。

### F. 写真選択のキャンセル・読込失敗

写真選択をキャンセルして再選択する。対応外ファイルでは、再試行できる日本語の案内を確認する。

合格: 白画面や無反応にならず、同じJPEG/PNGを選び直して進める。

### G. 通信切断からの復帰

写真保存または配達中に通信を切り、失敗表示後に通信を戻して再試行する。

合格: 同じ写真で再試行でき、二重登録されない。成功していないのに成功表示を出さない。

### H. 届いた写真の読込失敗

通信が不安定な状態で開封し、失敗表示から再読み込みする。

合格: 「写真をもう一度読み込む」が使え、操作不能にならない。

### I. アカウント作成・ログイン

オンボ完了後、設定からアカウント作成またはログインを行う。キャンセル後に再試行する。

合格: 匿名時の猫、送った写真、届いた写真を失わず、ホームへ戻れる。

### J. 無効なつづきリンク

期限切れ・使用済みの引き継ぎリンクを開く。

合格: 理由が日本語で表示され、復元済みならホームへ、復元できなければやり直しへ進める。

### K. 旧2枚目URL

`/home?from=onboarding_second_photo` と、旧 `next=second_photo` 付きのオンボ完了・アカウント作成URLを開く。

合格: 2枚目の要求やエラーを出さず、現行のホームまたはアカウント作成導線へ静かに正規化される。

## 4. 判定

- AまたはBに未解決の失敗があれば投稿を順延する。
- 写真の消失、別人の写真表示、共有範囲の誤り、二重登録、配達不成立、操作不能はP0とする。
- CのPWA案内だけが不調でもブラウザ利用が成立する場合はP1だが、iPhoneまたはAndroidの両方で案内が出ない場合は公開前に原因を確認する。
- 軽い表示ずれは証跡を残し、保存・配達・復元が成立する場合だけ初週内へ送る。
