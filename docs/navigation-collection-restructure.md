# 今日 / コレクション / ねこ 再整理方針

## 1. 結論

にゃるほどを以下の3タブ構成に整理する。

- 今日
- コレクション
- ねこ

それぞれの役割:

今日:
今日の猫をのぞく。いま見えたままを残す。診断ではなく、まず観察と入力に集中する。

コレクション:
見つけたもの、ポーズ、写真、時間帯の傾向、思い出、変化を見る。猫の暮らしがたまっていく場所。

ねこ:
猫プロフィール、毛色、アバター、追加、名前変更などを管理する場所。

## 2. プロダクト再定義

Before:
猫の様子を記録・診断するアプリ

After:
猫を見ていたら、暮らしがたまっていくアプリ

入力:
いま何してる？

蓄積:
猫コレクション

出力:
猫の暮らし分析 / 変化 / ふりかえり

にゃるほどは、猫の正解を当てるアプリではなく、飼い主の迷いを減らすアプリである。この前提は維持しつつ、日常的に開く理由を「困った時の診断」だけに寄せず、猫の暮らしが自然にたまる体験へ広げる。

## 3. 今日タブの役割

今日タブは「入力・観察」の場所。

置くもの:

- 猫カード
  - 猫名
  - 理解度
  - さいきんの{猫名}
  - いまの{猫名}
  - {猫名}の1日ミニマップ
  - 写真またはアバター
- いま何してる？入力カード
  - いつもの様子
  - ちょっと気になる
- 最近みつけたの軽い表示
  - コレクションへの入口として小さく出す

今日タブでやらないこと:

- 詳細分析を詰め込む
- 図鑑詳細を全部見せる
- 猫プロフィール管理をする
- 診断結果画面へ強く遷移させる

## 4. ホーム猫カードの方向

猫カードは「今日の猫をのぞく」場所。

表示内容:

- 猫写真またはアバター
- 猫名
- 理解度
- さいきんの{猫名}
- いまの{猫名}
- {猫名}の1日

方針:

- 写真がある場合は写真を主役にする
- 状態アイコンを増やしすぎない
- 状態は、さいきん / いま / 1日ミニマップで表現する
- 写真、アバター、状態バッジを同時に重ねすぎない
- 表示用アイコンは減らす
- 入力用アイコンは残す

## 5. いま何してる？の再定義

現在分かれている以下を統合する。

- いまどうしてる？
- ちょっと気になる？

変更後:
1つの入力カードに統合する。

カード見出し:
{猫名}はいまどうしてる？

補足:
いま見えたまま、ひとつでOK

中のセクション:

いつもの様子

- ねてる
- 毛づくろい
- 遊んでる
- ごはん
- トイレ
- ゴロゴロ

ちょっと気になる

- 鳴いてる
- ついてくる
- そわそわ
- 元気ない
- ケンカ
- よくわからない

保存方針:

- いつもの様子は `current_state` として保存
- ちょっと気になるは `concern` として保存
- ただし、いったん診断ページへは遷移しない
- 診断ロジックは削除せず、将来の詳しい相談導線として残す

## 6. 診断の扱い

診断は一旦ホームの主導線から外す。

方針:

- `/diagnose` は削除しない
- 診断ロジックも削除しない
- `feedbacks` 等の保存処理も削除しない
- ただし、通常のホーム入力からは `/diagnose` へ遷移しない
- 気になる様子はまず `concern` として `events` に保存する
- 将来「詳しく見る」「相談メモ」「受診メモ」として戻せるようにする

理由:
今は診断体験よりも、「猫の暮らしがたまる」体験を優先するため。

## 7. コレクションタブの役割

コレクションタブは「たまったものを見る場所」。

役割:

- 見つけたポーズを見る
- 写真や思い出を見る
- 最近の傾向を見る
- 変化を見る
- 猫ごとの暮らしの蓄積を感じる

画面構成案:

上部:

- コレクション
- {猫名}との“みつけた”を集める

最近みつけた:

- へそ天
- 箱入り
- のびー

ポーズコレクション:

- へそ天
- ごめん寝
- 箱入り
- 香箱
- 液体化
- のびー
- 窓辺監視
- ふみふみ
- おててないない
- 顔だけ出す
- まるまり
- すりすり

このごろ:

- 夜はへそ天が増えがち
- 最近は窓辺監視が多め
- 5/10 はじめてのへそ天

注意:

- コンプリート圧を強くしない
- 「達成」「未達成」ではなく「みつけた」に寄せる
- 写真がない日でも使える構造にする
- 写真は感情価値、ワンタップ記録は継続価値として両方残す

## 8. ねこタブの役割

ねこタブは「猫の管理」の場所。

置くもの:

- いま見ている子
- 猫一覧
- 猫追加
- 名前変更
- 毛色選択
- アバター表示

将来置くもの:

- 誕生日
- 性別
- 体重
- 家族共有
- 記録を残しておく
- 端末引き継ぎ

ねこタブから外すもの:

- ポーズ図鑑
- 思い出
- 詳細分析
- 最近の変化

## 9. 写真とアイコンの扱い

写真を追加する場合の方針:

写真:

- うちの子感
- 思い出
- コレクション
- 最近みつけた

アイコン:

- 入力タイルに使う
- 押すためのUIとして使う
- ホーム猫カード上の表示用アイコンは増やしすぎない

判断ルール:

- 操作用アイコンは残す
- 表示用アイコンは写真があるなら減らす

残す:

- いま何してる？の入力タイルアイコン
- コレクション内の写真サムネイル
- 必要最小限のミニマップ表現

減らす:

- 状態バッジ
- 写真に重なる小アイコン
- アバター + 写真 + 状態アイコンの三重表現

## 10. Pro機能との関係

無料:

- 今日の記録
- 基本コレクション
- 毛色別アバター
- 簡易ミニマップ

Pro候補:

- 写真からポーズ候補
- うちの子専用アバター生成
- 思い出タイムライン拡張
- 去年の今日
- 週次まとめ画像
- 家族共有
- 病院用レポート
- 長期保存

有料化の理由は「AI診断が見られるから」ではなく、以下に寄せる。

- この子の記録をずっと残せる
- 写真と思い出がたまる
- 家族と共有できる
- 最近の変化がわかる
- 病院で説明しやすい

## 11. 実装順序

Phase N1:

今日タブの入力カード統合

- いまどうしてる？ + ちょっと気になる？を1カード化
- concern 押下時に `/diagnose` へ遷移しない

Phase N3:
コレクションタブの最小UI

- まだ写真アップロードなし
- 固定ポーズカテゴリ
- 最近みつけた / ポーズ / このごろ

Phase N4:
ねこタブを管理専用として微調整

Phase N5:

- 写真なし手動追加
- 将来写真追加
