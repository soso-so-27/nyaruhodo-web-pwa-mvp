# ねてるねこ 先行期マーケティング運用計画 v0.1

作成日: 2026-07-18
対象期間: 先行30人が初週を終えるまで
上位規範: `docs/MARKETING-CANON.md`
計測規範: `docs/BETA-LEARNING-PLAN.md`

> 本文書は発信のトーンを変えるものではなく、獲得経路と判断手順を固定する運用文書。
> 投稿文の最終判断は中西が行い、数値は外向けには使わない。

## 0. 今回の判断

- 先行30人を、一度の告知で集める前提をやめる。
- **5〜10人の小さな波を3〜4回つくり、各波で「届く」までの成功と翌日の再利用を見てから次へ進む。**
- 2026-07-18のThreads単独投稿の弱さは、現時点では需要の否定ではなく、配信経路の実験失敗として扱う。
- Instagram、Stories、Threads、DMを同じ告知先として扱わず、それぞれに一つの役割を持たせる。
- 集客量より、最初の一通が壊れずに届き、翌日もう一枚入れたくなるかを先に検証する。

## 1. 判断根拠

### 自アカウントの実績（最優先）

- Instagram由来でThreadsにも展開された通常の猫投稿とアプリ構想投稿は、大きな閲覧・反応を得た。
- 2026-07-18のThreads単独投稿は、投稿約3時間時点で293表示、いいね4、返信・再投稿0。`src=threads` のアプリ流入も観測されなかった。
- 2026-07-18 15:14 JST時点のプロダクト記録では、入口11識別ID、写真選択4、初回保存3、初回到着2、初回開封2、翌日の一枚0。
- 同時点の入口別記録では、`instagram_story` は入口7・保存2・開封2、`instagram_bio` は入口3・保存1・開封0、`threads` は0。
- これらの識別IDは実人数ではない。ブラウザや環境をまたぐと同一人でも複数IDになるため、絶対人数ではなく経路比較と段階間の欠落に使う。

### 解釈

1. 現時点で、開始から開封までの実績が確認できる主経路はStories。
2. Instagram由来のThreads展開が強かった事実はあるが、「Instagram経由なら必ず優遇される」とは断定しない。自アカウントで再現性のある配信形として採用する。
3. 初回開封は発生しているが、翌日の再利用はまだ観測できていない。ここで流入だけを増やしても、価値仮説の確認にはならない。

### プラットフォーム側の補助根拠（二次）

- Threads公式は、返信が閲覧の大きな部分を占め、会話を生む投稿が推薦されやすいと説明している。写真だけでなく、写真に文脈を添えることも推奨されている。
  https://about.fb.com/news/2024/10/find-your-community-with-new-threads-educational-insights/
- Threadsのトピック付与は一般に閲覧増と相関し、投稿がInstagramやFacebookに推薦される場合もある。ただし、個別投稿の配信を保証するものではない。
  https://about.fb.com/news/2025/03/new-threads-features-more-personalized-experience-you-control/amp/
- InstagramのTrial Reelsは、非フォロワーに先に出して24時間後の反応を見る用途に使える。これは継続仮説が見えた後の認知実験に限る。
  https://about.fb.com/news/2024/12/trial-reels-try-content-non-followers-first-see-what-perfoms-best/

## 2. 先行期の目的

目標は「30人がリンクを開くこと」ではなく、**30人が一度は写真を入れ、ねこだよりを受け取り、ひらくこと**。

その上で、次の三段階を別々に見る。

| 段階 | 問い | 主な指標 |
|---|---|---|
| 獲得 | 届くべき人に説明が届いたか | 入口、写真選択開始 |
| 初回成立 | 約束した一通が壊れず届いたか | 写真保存、初回到着、初回開封 |
| 継続 | 翌日も撮る理由になったか | 翌日の一枚、初週5日以上 |

## 3. チャネルの役割

| チャネル | 役割 | 運用 |
|---|---|---|
| Instagramフィード | 信頼と起源の物語 | あめ・むぎが主役。猫の日常7、アプリ3を維持。アプリ投稿も猫の実体験から始める |
| Instagram Stories | **開始への主導線** | 1〜2枚で完結。直接リンクを置く。連投せず、1回ごとに24時間観察 |
| InstagramからのThreads展開 | 到達範囲の拡張 | 重要なフィード投稿は当面Threads連携を既定にする。自アカウントの実績ある形を優先 |
| Threads単独投稿 | 会話仮説の小さな実験 | 重要告知を任せない。週1回以下。リンク獲得ではなく返信・会話の有無で判定 |
| DM | 温度のある人への手渡し | 過去に「使いたい」と明示した人、質問した人に一度だけ知らせる。新規の営業DMや追跡はしない |
| bioリンク | 常設の受け皿 | いつでも開始できる状態を保つ。投稿ごとの成果と混同しない |
| Reels / Trial Reels | 将来の認知拡張 | 翌日・初週継続が見えてから。日常の猫動画として試し、アプリ説明を主役にしない |

`docs/july-campaign-copy-set-v1.2.md` と `docs/instagram-post3-final-2026-07.md` の
「Threads自動シェアOFF」は、完了した投稿3の実施記録として残す。2026-07-18以降の新しい発信は
本計画を優先し、重要なInstagramフィード投稿はThreads連携を既定とする。

### sourceパラメータ

- Stories: `src=instagram_story`
- bio: `src=instagram_bio`
- DM: `src=instagram_dm`
- Threads: `src=threads`
- その他の手渡し: 用途が分かる固有値を付ける

リンクを使う発信は、公開前にsourceが保持されることを一度確認する。

## 4. これから14日間の進め方

### 直近24時間

- 2026-07-18のThreads投稿は消さず、追記・再投稿もしない。24時間値を記録して実験を閉じる。
- 新しい公開告知は重ねない。
- 初回開封済みの人が翌日もう一枚入れるかを観察する。

### 第1波: Storiesと温度のあるDM（5〜10人目安）

- Storiesを1〜2枚だけ出す。1枚目はあめ・むぎの具体的な寝顔、2枚目に短い仕組みとリンク。
- 過去に「使いたい」と言った人へ、完成を一度だけDMで知らせる。返答や利用を求めない。
- 投稿後24時間は文言・導線を変えず、`story` と `dm` を分けて観測する。

### 間の2〜3日

- 通常の猫投稿へ戻る。アプリの話を続けて重ねない。
- InstagramフィードからThreadsへの連携はオンにし、普段の配信形を回復する。
- 初回利用者の到着、開封、翌日の一枚、真のエラーだけを確認する。

### 第2波: 使った瞬間の報告

- 第1波で保存者全員への到着が成立し、開封の技術エラーがなく、翌日の一枚が少なくとも3人に1人程度あれば実施する。
- 「アプリを作った説明」ではなく、あめ・むぎと実際に使った一場面を中心にする。
- Webアプリであること、最初はあめ・むぎが届く場合があることを小さく正直に添える。
- Instagramフィードを起点にし、Threadsへ連携する。

### 第3波: 必要な場合のみ

- 30人に届いていなければStoriesまたは温度のある手渡しをもう一度行う。
- 同じ説明を繰り返さず、「撮る前」「20時」「ひらいた後」のうち、まだ見せていない一場面だけを出す。
- 一つの波で変えるのは、写真・一行目・チャネルのいずれか一つにする。

## 5. 投稿設計の型

1. **猫の具体的な一場面**: 誰の、いつの、どんな寝顔か。
2. **小さな発見**: あとから見ると、その頃まで思い出す等。
3. **ねてるねことの接続**: その体験から作ったことを短く。
4. **期待値の調整**: Webアプリ、始まったばかり、最初はあめ・むぎが届く場合がある。
5. **一つだけの入口**: リンクまたはbio。複数の操作を求めない。

長い思想説明、機能一覧、利用のお願いを一つの投稿に同居させない。

## 6. 判定線

### 壊れていないことの条件

- 写真を保存した識別IDには、初回ねこだよりが100%届く。
- 届いたねこだよりを開こうとしたとき、100%正常にひらける。未開封のまま置く行動は障害に数えない。
- 同じ写真が二重に「おくった」「とどいた」へ入らない。
- 実ユーザーに同じ未解決P0が複数発生したら、次の告知を止める。

### 10件ごとの診断トリガー

小標本の合否ではなく、次に見る場所を決めるための線とする。

| 観測 | 診断 |
|---|---|
| 入口10に対し写真選択開始3未満 | 対象・説明・安心材料の問題を先に見る |
| 写真選択開始10に対し保存8未満 | 写真選択、LINE内ブラウザ、文言、待ち時間を調べる |
| 保存10に対し到着10未満、または開封操作に失敗 | P0。集客を止めて配達・開封を直す |
| 開封10に対し翌日の一枚3未満 | 集客ではなく、ホームと翌日の理由を改善する |

### 30人到達時の判断

`BETA-LEARNING-PLAN` の「初週5日以上」を正とする。

| 初週継続 | 判断 |
|---|---|
| 12人以上 | 事前予想の下限達成。P3準備へ進む |
| 6〜11人 | 先行期を継続。一週に一つだけ改善して再計測 |
| 1〜5人 | 核心体験を再設計。新規獲得を広げない |
| 0人 | 配達が技術的に成立していたか再監査後、休止・転換を含めて判断 |

一つの投稿、一日、または30人未満の母数で撤退判断はしない。

## 7. 数字の読み方

- **表示が少なく、入口もない**: 配信・チャネルの問題。
- **表示はあるが入口がない**: 一行目、写真、説明、リンクの問題。
- **入口はあるが写真選択がない**: 猫を飼っていない、今写真がない、安心できない、説明が長い等を切り分ける。
- **写真選択はあるが保存されない**: UIまたは技術の問題。
- **保存されたが届かない**: P0。マーケティングで解釈しない。
- **開封されたが翌日戻らない**: プロダクトの継続価値の問題。

「率が低い」を一括で語らず、必ずどの段階で止まったかを先に言う。

## 8. 発信前カード

各発信の前に、次の5行だけを埋める。

```text
目的:
チャネル:
source:
今回変える一つ:
判定時刻と、次にすること:
```

- 公開後24時間は、安全上の問題を除き、削除・再投稿・文言変更をしない。
- 結果が弱くても、その場で別チャネルへ同じ告知を重ねない。
- 自アカウントの過去実績を、一般論より優先する。

## 9. 当面やらないこと

- 広告出稿
- Threads単独投稿を主要獲得路にすること
- アプリ告知の連投
- 利用者への確認DM、再利用の催促
- 他ユーザーの届いた写真や感想の無断掲載
- 初回継続が不明なままリールで広く集客すること
- 一つの投稿結果で需要や撤退を断定すること
