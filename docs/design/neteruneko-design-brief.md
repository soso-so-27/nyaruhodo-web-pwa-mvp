# ねてるねこ ホーム再設計ブリーフ(Claude Design持ち込み用)

添付: neteruneko-home-v3-desk-model.html(現時点の到達点。これを出発点に視覚を追い込む)

## 1. 製品と前提(60秒で読める版)

「ねてるねこ」は猫の寝顔を1日1枚交換するiOS向けPWA。朝〜日中に自分の猫の寝顔を撮ると、**夜8時ごろ、どこかの誰かの猫の寝顔が1枚届く**。手紙の比喩で統一された世界(あずける・封筒・そっとひらく)。プレローンチのβ、開発者は1人、ユーザーの猫は「むぎ」。

製品テーゼ: 寝顔を撮る習慣が「自分の猫を見る回数」を増やし、愛着を深める。**主役は常に自分の猫と、受け取る喜び。送信・共有はほぼ不可視の配管であるべき。**

## 2. 確定済みのデザイン判断(ここは再審理しない)

1. **見出しの廃止**: 「おあずかり中」「きょうの 2まい」等の明朝大見出しは全状態で存在しない。状態は絵が語る
2. **机モデル**: ホーム=「きょうのフレーム(自分)+夜の手紙(相手)」の2スロット。**何も移動しない**(輸送・矢印・進捗バーの表現は禁止。撮った写真が誰かへ送られる物語を視覚化しない)
3. **ホーム=アルバムの第1行の舞台化**: 開封後(state4)はアルバムのきょうの行と同じ文法(2タイル+α)。アルバム側は逆に静音化(台帳: 全行同サイズ・演出なし)
4. **丸ボタン廃止**: 撮影の入口は「呼吸する空のフレーム」(写真と同じ角丸正方形。形=affordance)
5. **コピーの段階表示**: はじめの7日(または交換5回)だけ小さな一行が添えられ、以後消える。完全な説明は設定内「使い方」ページが常設で持つ
6. **「そっと」は実装する**: 開封は封筒の長押し。おさえているあいだだけ写真が現像され、途中で離すと閉じる。ラベルは「おさえて ひらく」

## 3. コピー監査の結果(審査基準: 機能→ブランドの順)

- 残留: 「とっておく」/「むぎ、ねてる?」(=猫を見ろという機能的プロンプト、week1のみ)/「おさえて ひらく」/「きょうも、◯ひきの ねこが ねています」(気配の一行、最小・常設)/猫名ラベル(アルバムのみ)
- 削除: 「おあずかり中」「ねがおが とどいています」「きょうの 2まい」「また、ねていたら。」「そっとひらく」(→長押し化で「おさえて ひらく」へ)
- 原則: **言葉が約束する手触りをUIが実装できないなら、その言葉を削る。** 副詞は実装するか削除
- ラベル「むぎ/どこかのこ」はホームでは訓練輪=week1のみ。自分の猫に名札は不要
- 文体: ひらがな多め・分かち書き・敬体最小。新しいコピーを足す場合もこの文体

## 4. トークン

- 背景: #f7f3ea(朝のクリーム)。時間で暮れる: state3〜4は #f4ebe1→#ebdcd3 系へ
- カード: #fffdf8 / インク: #4a4338 / サブ: #9a9183 / 罫線: #e7dfd0
- アクセント(1色のみ): ローズ #c2607c(封筒・届きの色)。濃: #a84e68 / 淡: #f5e3e8
- 写真の言語: **角丸正方形タイル**(白フチ8px、radius 24/17)。アプリ内の写真は例外なくこの形
- 手紙の言語: **横長(約3:2)・角丸・わずかに傾く(-2〜-3°)「物」**。スロットを満たさない
- 書体: 見出し用の明朝は原則使わない(廃止済み)。UI文字は -apple-system / Hiragino Sans、小さく・薄く
- 影: 最小(カードに柔らかい1段のみ)。フラット基調

## 5. 4状態+1bの仕様

- **state1(朝・未撮影)**: 呼吸する空フレーム(scale 1→1.016、4.2s)+淡い手紙(opacity .55)+下に「きのうの2まい」ミニ。タップ領域=フレーム全体→カメラ
- **state2(あずけてある)**: フレームに自分の写真(動かない)。手紙は(1)中身が時刻に比例して満ちる=満水20時(2)わずかに育つ scale .94→1(3)背景が暮れる。**手紙タップで「よる8じごろ」が2秒だけ表示**(言葉は聞かれたら答える)
- **state3(20時以降・未開封)**: 手紙が大きくなり(188×130)、封蝋。自分の写真は小さく退く。長押しで現像→完了で開いたまま
- **state4(開封後)**: 2タイル等サイズ+「どこかのこ」ラベル(week1のみ)。完成のサイン=揃った2枚そのもの
- **state1b(20時以降・未撮影)**: 手紙は満水のまま封が閉じた表示。フレームの呼吸は続く(明日の分として撮れる)。week1のみ「いまとると、あしたのよるに とどく」

## 6. Claude Designで探索してほしいこと(優先順)

1. **state1のバリエーション3案**: 現状まだ垢抜けない。空フレームの中身(カメラグリフ/猫の気配/無)、きのうミニの距離と濃度、手紙との間合い。「埋めたくなる未完成」を保ったまま品を上げる
2. **手紙の造形**: 比率・傾き・フラップの表現・封蝋の有無。スキューモーフィズムは「机に置かれた物」と感じる最小量(Locket程度)。やりすぎ=安っぽい
3. **実写真との共存**: タイルに実際の猫写真(騒がしい・色とりどり)を入れた状態で、白フチと淡い手紙が負けないか。プレビューには実写を必ず流し込むこと
4. **state2の生気**: 「静かすぎて壊れて見える」リスクの回避。満ちのシマー、ごく稀な微粒子等、calmを壊さない範囲の生命感
5. **時間で暮れる背景**: 朝→夜のグラデーション設計(4状態の背景が1日の連続に見えること)

## 7. 制約

- アクセシビリティ: 文字を消した要素は全てaria-labelで同等の説明。長押しはタップ開封のフォールバック必須。prefers-reduced-motionで全アニメーション停止
- 状態の判定可能性: 各状態はdata-state属性で機械的に識別できること(E2Eが検証する)
- ボトムナビ: きょう / アルバム / ねこ の3タブ(「とる」から改名済み)。きょうのアイコンは2タイルのミニチュア(空き=点線)
- 1人開発・実装はNext.js PWA。凝った3Dや重いアセットは不可。CSS/SVGで成立する範囲

## 8. 未決の論点(議論の相手になってほしい)

- 「おさえて ひらく」vs「そっと ひらく」: 長押し実装により「そっと」は機能の裏付けを得た。どちらの語がこの世界に合うか
- 「また、ねていたら。」: 機能ゼロの純ブランド。削除済みだが、結びの挨拶として常設する選択肢は残っている
- 気配の一行の深夜帯(0時すぎの「きょうも」)の扱い
